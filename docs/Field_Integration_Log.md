本文档记录 2026-03-10 这一轮实物接线、烧录、联网与传感器排障的现场结论。它不是“理想接线图”，而是当前桌面实测进度与问题台账。

## 2. 已验证通过的现场结果

1. `PlatformIO` 设备侧构建成功，且已通过 `ST-LINK` 完成多次烧录。
2. OLED 在烧录并正确接线后可以点亮。
3. ESP32 AT 固件已重刷，默认 AP 为 `Aquarium_Setup`，固定密码为 `12345678`。
4. DS18B20 在补上 `4.7kΩ` 上拉并重新压紧接线后恢复正常，不再固定停在 `85°C`。
5. 水位传感器已能区分“离水”和“浸水”，当前代码里做了演示向标定。
6. 设备曾实测进入 `NET=ONLINE`，并在华为云控制台出现过 `ONLINE` 状态。

## 3. 本轮联调中确认的问题与挫折

### pH 模块

- 先后测试过自来水、白醋、BNC 短接等办法，读数长期停在 `2.5 ~ 2.9` 左右，偶发跳到 `14`、`9.x`、`5.x`，表现不稳定。
- 将模块供电从 `5V` 改到 `3.3V` 后，读数有变化，但仍不可信。
- 按商家建议调节靠近 `BNC` 的电位器多圈后，仍未得到可复现的正常响应。
- 当前判断：`pH` 模块或电极链路高度可疑，已按“疑似坏件/需售后”记录，后续应优先换货再测。

### TDS 模块

- 按 `AO/A1`、`3.3V/GND` 接好后，空气中和水中都曾长时间读 `0`。
- 中途出现过一次约 `1623` 的读数，但不能稳定复现。
- 当前判断：`TDS` 模块链路尚未打稳，既可能是模块/探针本体问题，也可能是商家版本与文档不一致，已记录为“待与商家联调视频复核”。

### 浊度模块（TS-300B）

- 这一轮被搁置，尚未形成实测闭环。
- 原因不是代码已确认没问题，而是现场优先级让位给联网、OLED、温度、水位四条主线。

### 水位传感器

- 这一路已经能工作，但“液位高度 -> 百分比”的曲线明显不线性。
- 现场出现过“刚浸入一部分就接近 100%”和“浸入一半仍接近 0，随后突然跳满”的情况。
- 用户最终接受当前版本作为演示版，因此当前水位标定只能视为“可演示”，不能视为“精确测量版”。

### AP 配网 / OLED 现场体验

- iPhone Safari 在访问 `http://192.168.4.1/` 或直接打 `/config?...` 时，出现过白屏或“已丢失网络连接”。
- 最近一轮诊断版固件中，OLED 右上角页码曾长期停在 `P1`，说明页面轮播或主循环仍需继续排查。

## 4. 当前适合继续实测的最小闭环

当前最稳的实物组合是：

1. `NUCLEO + OLED + ESP32 + DS18B20 + 水位传感器`
2. 先验证 `OLED显示 -> Wi-Fi上线 -> 云端在线/影子刷新`
3. 再追加 `pH/TDS/浊度`，不要一次把所有可疑模块都挂上
